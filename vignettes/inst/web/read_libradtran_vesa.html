Read libRadtran output file.

Usage

read_libradtran_vesa(file, date = NULL, geocode = NULL, label = NULL, tz = NULL, locale = readr::default_locale(), simplify = TRUE)

Arguments

file
character string
date
a POSIXct object, but if NULL the date stored in file is used, and if NA no date variable is added
geocode
A data frame with columns lon and lat.
label
character string, but if NULL the value of file is used, and if NA the "what.measured" attribute is not set.
tz
character Time zone is by default read from the file.
locale
The locale controls defaults that vary from place to place. The default locale is US-centric (like R), but you can use locale to create your own locale that controls things like the default time zone, encoding, decimal mark, big mark, and day/month names.
simplify
logical Remove redundant columns from returned object.

Value

a source_spct object, possibly containing several spectra in long form and a datetime column.

Description

Reads and parses the header of a text file output by libRadtran for a solar spectrum simulation to extract the header and spectral data. The time and date fields are converted into a datetime object.

Note

Tested only libRadtran version 2.0
